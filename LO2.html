üìé View Drawing Page

How it works:
This function paints the whole background of the canvas with a smooth color gradient. It blends from pink to light green as you move across the screen.

Why I used it:
I wanted the background to feel colorful and alive instead of just a solid color. It gives the page more energy and personality.

Where I learned it:
I found how to use createLinearGradient() from W3Schools and some examples I saw on CodePen when exploring creative background effects.

What it does:
This function makes everything on the screen come alive. It keeps running over and over, updating the background and the glowing mouse trail every moment.

Step by step:
- First, it clears the old stuff from the screen.
- Then it paints the colorful gradient background again.
- If the mouse has moved, it adds a glowing dot where the mouse is.
- It slowly makes the old dots fade away.
- Then it draws all the glowing dots that are still visible.
- Finally, it repeats the whole process, so it keeps running like an animation.

Why I used it:
Without this, nothing would move. This is what makes the glowing trail follow the mouse and the whole page feel interactive and alive.

How I learned it:
I saw examples online where people used requestAnimationFrame() to create smooth animations, and I wanted to try it myself!

üìé View Rainbow Flow Page
I had made mouse interactions before, but the new challenge was the background. I thought adding color would be easy, but it turned out to be much more complex than expected.

The background is created using WebGL shaders, which lets me control how every single pixel looks. There are two parts to it:
Vertex Shader
Just sets up the screen area for drawing and it tells the browser: ‚Äúfill the whole canvas.‚Äù

Fragment Shader
This is where the magic happens. The fragment shader uses math to:
- Make rainbow colors that smoothly change over time.
- Add swirl and wave effects so the background looks liquid-like and always moving.
- React to the mouse, making it glow more where the cursor is.

It runs on the canvas, and updates every frame (around 60 times per second), which is why the background looks alive and animated.
